# CLAUDE.md - Project Guidelines

## Project Overview
This is a test repository for Claude Code GitHub Actions automation. Claude can be invoked on PRs and issues by tagging @claude.

## Code Style
- Use clear, descriptive variable and function names
- Write concise comments for complex logic
- Follow existing patterns in the codebase

## PR Review Guidelines
- Check for potential bugs and edge cases
- Suggest performance improvements where applicable
- Ensure code follows project conventions
- Verify tests are included for new features

## Common Commands
- `npm install` - Install dependencies
- `npm test` - Run tests
- `npm run build` - Build the project

## Important Notes
- Always run tests before submitting a PR
- Keep commits atomic and well-described
- Do not commit sensitive data (API keys, secrets, etc.)
